# Drone-Autonomous-Flight-Competition
MATLAB을 이용한 미니 드론 자율비행 및 제어 알고리즘 개발

이 대회는 MATLAB에서 주관하는 미니 드론 자율주행 시뮬레이션 대회로 1차 예선은 시뮬레이션으로 알고리즘을 판단하고 최종 예선 때 직접 임베디드 보드에 우리가 만든 MATLAB Simulink 블록을 코드로 최적화 해 실제 드론으로 테스트하는 대회입니다.

## 실행 방법 

시뮬레이션을 정상적으로 구동하기 위해 아래 순서대로 실행해 주세요.

1. MATLAB을 실행한 후, 전체 파일 중 **`MinidroneCompetition.prj`** 파일을 먼저 실행합니다.
2. 프로젝트가 로드되면 **`variable.m`** 파일을 실행합니다.
3. 설정이 완료되면 최종적으로 Simulink에서 실행을 누르시면 됩니다.

## 알고리즘 설명: 다중 ROI 기반 동적 Pure Pursuit 알고리즘 (일명 와이파이 알고리즘)

기존 자율주행에서 자주 쓰이는 **Pure Pursuit** 알고리즘은 전방의 고정된 하나의 지점(Look-Ahead Point)만 주시하기 때문에, 급격하게 꺾이는 직각 구간이나 중급커브에서 경로를 이탈하거나 비행이 불안정해지는 한계가 있었습니다. 이를 보완하기 위해 다음과 같은 알고리즘을 설계했습니다.

1. **다중 관심영역(ROI) 설정 및 중심점 도출**
   * 드론 하단 카메라 영상에서 중심점을 기준으로 거리가 각기 다른 3개의 선형 띠를 관심영역(ROI)으로 지정합니다.
   * 각 띠 안에서 라인(빨간색 영역)의 중심점을 실시간으로 검출하여 인지합니다.

2. **동적 주시점 제어 및 경로 곡률 판단**
   * 기본적으로 드론은 중간 위치의 띠가 검출한 중심점을 타겟으로 추적하며 안정적으로 주행합니다.
   * 이때, 3개의 띠가 검출한 포인트들의 상대적인 위치 관계를 분석하여 현재 따라가고 있는 빨간 길의 **곡률(Curvature)**을 판단합니다.

3. **Pure Pursuit 한계 극복 (급커브 대응)**
   * 곡률을 분석하여 급격한 회전이나 직각으로 꺾이는 구간(곡률이 임계값 이상인 경우)을 감지하면, 전방 주시 거리를 동적으로 축소합니다.
   * 즉, 가장 멀리 있는 점 대신 **드론과 가장 가까운 첫 번째 띠의 중심점**을 추적하도록 제어 명령을 전환함으로써, 급커브에서도 오버슈트(Overshoot) 없이 안정적으로 궤적을 추종하도록 보완했습니다.

## 느낀점

초기에는 자율주행 경로 추종에 가장 널리 쓰이는 기본 **Pure Pursuit** 알고리즘을 적용하여 드론을 제어했습니다. 하지만 고정된 전방 주시 거리(Look-Ahead Distance)의 한계로 인해, 급격한 곡선이나 직각으로 꺾이는 구간에서는 드론의 속도를 아무리 감속하더라도 궤적을 이탈하거나 비행이 극도로 불안정해지는 고질적인 제어 문제가 발생했습니다.

이 문제를 해결하기 위해 조향 매커니즘을 깊이 고민한 끝에, Pure Pursuit의 기본 틀은 유지하되 **전방 주시점(Look-Ahead Point)을 다중(3개 영역)으로 확장하는 아이디어**를 구상했습니다. 

단일 점이 아닌 3개의 띠를 통해 주행 경로의 곡률을 실시간으로 조기에 탐지하고, 곡률의 심한 정도에 따라 추종하는 주시점을 동적으로 변경(Look-Ahead 변경 기반 속도 및 조향 적응형 제어)하도록 알고리즘을 개선했습니다. 

기존 알고리즘의 명확한 한계를 스스로 분석하고, 수많은 시뮬레이션 끝에 유연하고 안정적으로 급커브를 통과하는 드론을 구현해 냈을 때 정말정말 뿌듯했고 큰 성취감을 얻을 수 있었습니다. 이론으로만 접했던 제어 이론을 실제 문제 상황에 맞춰 창의적으로 보완하고 발전시켜 볼 수 있었던 값진 경험이었습니다.
